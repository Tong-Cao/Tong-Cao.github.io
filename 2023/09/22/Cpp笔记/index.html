C++

  • 输出Hello World
  • 双冒号作用域运算符
  • 命名空间namespace的应用：
    • 用于解决命名冲突的问题
    • 必须在全局作用域下声明
    • 命名空间下可以放入函数、变量、结构体、类
    • 命名空间可以相互嵌套
    • 命名空间为开放式，可随时加入新成员
    • 匿名命名空间相当于 空间中的变量加入static 成为静态变量

  • C++对C语言的增强：

    • 全局变量检测增强
    • 函数检测增强（函数中必须有参数类型、返回值、传参数量必须匹配）
    • 类型转换增强（malloc返回void*，C可以不用强转，C++必须强转）
    • struct增强（C++允许添加函数，实例化可以不加关键字struct）
    • 三目运算符增强（a>b?a:b ，a>b时C返回a的值，C++返回变量a）
    • const（常量）增强（C++中const int a=10 ，不会为a分配内存，无法通过取地址改变a的值）

  • 引用：给变量起别名（必须有初始化，初始化后只能对应一个变量不能修改）

    引用传递：类似传地址

  • 类的设计：

  • 类的权限：

    • public 公有权限，类的里面和外面都可以访问
    • pravite 私有权限，类里面可以访问，外面不行
    • protected 保护权限，类里面和当前类的子类可以访问，外部不行

  • 内联函数：

    • 用来解决宏函数缺陷
    • 函数中存在循环语句、过多条件判断、对数据进行取址操作时编译器不会进行内联编译

  • 函数重载：

    • 函数名以及作用域相同，但是函数的参数个数、类型、顺序不同可以满足函数重载
    • 函数返回值不可以作为函数重载的条件

  • extern C

    • 解决在C++中调用C语言的代码问题

继承

  • 继承

  • 继承方式

  • 继承中的构造和析构顺序

    • 子类创建对象时，先调用父类构造再调用自身构造。
    • 析构顺序和构造顺序相反。
    • 子类不会继承父类的构造和析构函数
    • 如果父类中没有合适的默认构造，那么子类可以利用初始化列表的方式显示调用父类的有参构造

  • 继承中的同名处理

    成员属性：直接先调用子类，如果想调用父类中的需要加入作用域 （如：s1.Base::m_A 在子类实例s1上调用父类Base的m_A的值)

    成员函数：直接调用子类，父类所有版本都会被隐藏，除非加上作用域

  • 多继承

    class A ：public B1，public B2（从B1，B2 中继承）

    多继承可能会引起二义性（B1，B2相同命名冲突）， 一定要加上作用域

  • 菱形继承问题以及解决

    使用虚基类来继承，class A ：virtual public B （此时A中继承的数据和B共享同一份数据）

多态

  • 多态： 父类的引用或者指针指向子类对象

  • 多态分类

    • 静态多态：一般指函数的重载
    • 动态多态：在继承关系中 使用虚函数

  • 静态联编和动态联编

    • 静态联编：地址早绑定，编译阶段绑定好地址
    • 动态联编：地址晚绑定，运行时绑定好地址

  • 抽象基类与纯虚函数

    • 纯虚函数写法 virtual void func() = 0;

    • 如果父类中有纯虚函数，子类中必须实现具体的函数

    • 如果父类中有纯虚函数，此父类无法实例化对象

    • 通常称包含纯虚函数的类为抽象类

  • 虚析构和纯虚析构

    • 写法：在父类中将析构函数改为 virtual ~类名() {};

    • 用以解决通过父类指针指向子类对象时释放不干净的问题

    • 纯虚析构：virtual ~类名() = 0;

    • 纯虚析构需要声明且需要实现 类内声明，类外实现

    • 如果函数中出现纯虚析构函数，那么这个类也算抽象类（不可实例化）

类型转换

  • 静态类型转换

    语法 ：static_cast<目标类型>(原始数据)

    string 转 char* ：**.c_str();**

    可以进行基础数据类型转换、以及父与子类的相互转换

  • 动态类型转换

    语法 ：dynamic_cast<目标类型>(原始数据)

    • 不可以进行基础数据类型转换
    • 子类转向父类可以进行（安全转换）
    • 父类向子类转换时，只有发生多态才可以(有虚函数)

  • 常量转换

    语法 ：const_cast<目标类型>(原始数据)

    用于将数据转换为const类型或者反过来。

    只可以对指针和引用进行操作。

异常

  • 异常的基本使用

    • try{} 执行一段代码
    • 在代码中如果出现异常 使用throw抛出异常
    • 使用catch捕获异常

  • 栈解旋

    从try开始到throw抛出异常之前，所有栈上的对象都会被释放。这一过程称为栈解旋。

    栈上对象的构造顺序与析构的顺序相反（最先构造的对象最后一个析构）
